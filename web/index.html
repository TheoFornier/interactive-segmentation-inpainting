1. Introduction

Ce projet s'inscrit dans le cadre du cours de photographie algorithmique. L'objectif est de concevoir un outil interactif permettant à un utilisateur de :

  1. Sélectionner un objet dans une image via un rectangle de délimitation et/ou des coups de pinceau (foreground / background).
  2. Obtenir un masque de segmentation précis grâce à l'algorithme GrabCut (graph cut).
  3. Reconstruire automatiquement l'arrière-plan à la place de l'objet retiré, via un inpainting neuronal (LaMa) ou un fallback OpenCV.

L'ensemble est exposé dans une application desktop PySide6, avec un pipeline de traitement entièrement modulaire et des scripts d'évaluation quantitative.

Problématique principale : comment obtenir une segmentation robuste à partir d'annotations minimales de l'utilisateur, et comment reconstruire le fond de façon naturelle et cohérente avec le reste de l'image ?

3. Segmentation interactive (GrabCut)

Principe de GrabCut

GrabCut (Rother et al., 2004) est un algorithme de segmentation interactive par graph cut. Il modélise la distribution de couleur du premier plan et de l'arrière-plan par des Mélanges Gaussiens (GMM), puis résout un problème de coupe minimale dans un graphe (énergie de Gibbs) pour affecter chaque pixel à une classe.

Le processus itère en alternant :

  1. Mise à jour des GMM foreground / background à partir du masque courant.
  2. Résolution de la coupe min-cut / max-flow pour mettre à jour le masque.

Pipeline de segmentation

1

Initialisation

L'utilisateur dessine un rectangle autour de l'objet (GC_INIT_WITH_RECT) et/ou applique des scribbles foreground / background (GC_PR_FGD, GC_BGD). Les deux modes sont combinables.

2

Résolution GrabCut

Le solveur (graph_cut_solver.py) appelle cv2.grabCut deux fois : d'abord avec le rectangle (si fourni), puis avec le masque mis à jour (GC_INIT_WITH_MASK) pour 5 itérations par défaut.

3

Génération du masque binaire

mask_generator.py binarise le masque GrabCut en ne conservant que les labels GC_FGD et GC_PR_FGD comme foreground.

4

Raffinement morphologique

mask_refinement.py applique une ouverture (kernel 3×3), une fermeture (kernel 7×7), une dilatation (1 itération), puis ne conserve que la plus grande composante connexe contenant les seeds foreground.

5

Extraction de l'objet

L'objet est isolé sur fond noir (extracted_object_bgr) et le fond est mis à transparent via canal alpha (background_removed_bgr). Un canal de feathering est appliqué sur les bords du masque (Gaussian blur, σ = 11 px).

Modes d'initialisation supportés

🔲 Rectangle seul

L'utilisateur délimite l'objet par un rectangle. GrabCut initialise automatiquement les GMM. Rapide mais moins précis sur les objets complexes.

GC_INIT_WITH_RECT

✏️ Scribbles (PAP / PPP)

Pinceaux vert (foreground) et rouge (background). Permet d'affiner les zones ambiguës. Peut être utilisé seul ou en complément du rectangle.

🔲✏️ Rectangle + Scribbles

Combinaison optimale : le rectangle donne une initialisation grossière, les scribbles affinent les bords critiques. Mode recommandé.

4. Reconstruction du fond (Inpainting)

Stratégie générale

Une fois l'objet retiré, un masque de trou est construit à partir du masque de segmentation raffiné. Le pipeline calcule une ROI locale autour du trou pour n'envoyer qu'une région pertinente à l'algorithme d'inpainting, puis fusionne le résultat dans l'image originale.

Backend LaMa (principal)

LaMa (Large Mask inpainting) est un modèle de deep learning basé sur des Fast Fourier Convolutions (FFC) qui capturent les répétitions et textures à longue portée. Il est utilisé via le paquet Python simple-lama-inpainting.

🧠 Réseau neuronal FFT Convolutions Grandes masques Cohérence globale

Fallback OpenCV

Si LaMa n'est pas disponible, le système bascule sur cv2.inpaint avec la méthode TELEA (algorithme basé sur l'équation de transport rapide). Ce fallback est fonctionnel sur de petits trous mais moins convaincant sur de larges zones.

method = cv2.INPAINT_TELEA if config.INPAINT_FALLBACK_METHOD == "telea" else cv2.INPAINT_NS
cv2.inpaint(image_bgr, hole_mask, config.INPAINT_FALLBACK_RADIUS, method)

Pipeline d'inpainting

1

Construction du masque de trou

Le masque binaire de l'objet est dilaté (INPAINT_MASK_DILATION = 2) pour couvrir les artefacts de bord.

2

Calcul de la ROI locale

Une bounding box englobant le masque de trou est calculée avec une marge de 20 % (INPAINT_ROI_MARGIN_RATIO), minimum 24 px.

3

Inpainting sur la ROI

Seule la région recadrée est envoyée au backend (LaMa ou TELEA), réduisant la charge mémoire et le temps de calcul.

4

Fusion avec feathering

La ROI inpaintée est réintégrée dans l'image originale. Un flou gaussien (σ = 17 px) sur les bords du masque assure une transition douce (feathering).

5. Interface utilisateur (PySide6)

L'interface est une application desktop Qt développée avec PySide6. Elle expose un workflow linéaire guidé par numérotation des boutons.

Workflow utilisateur

1

Charger une image

Dialogue de fichier, chargement et redimensionnement automatique (max 1280×900 px).

2

Mode rectangle (optionnel)

Dessin d'un rectangle de délimitation sur le canvas.

3

Pinceau premier plan

Scribbles verts sur l'objet à segmenter.

4

Pinceau arrière-plan

Scribbles rouges sur le fond.

5

Lancer la segmentation

Exécution du pipeline GrabCut + raffinement.

6

Remplir l'arrière-plan

Exécution du pipeline d'inpainting (LaMa ou TELEA).

Modes d'affichage disponibles

Image brute chargée.
Annotations
Scribbles fg/bg superposés.
Overlay
Masque semi-transparent sur l'image.
Masque
Masque binaire coloré.
Objet extrait
Objet isolé sur fond noir.

Image finale après inpainting.

Autres fonctionnalités

  • Spinbox pour ajuster la taille du pinceau à la volée.
  • Bouton Réinitialiser annotations sans recharger l'image.
  • Bouton Sauvegarder résultats : écrit le masque, l'objet extrait, le fond inpainted et une planche comparative dans data/output/.
  • Log textuel en temps réel affichant les métriques de chaque étape.

6. Évaluation quantitative

Métriques de segmentation

MétriqueFormuleInterprétation
IoU (Intersection over Union)|P∩G| / |P∪G|Recouvrement entre masque prédit et vérité terrain. 1 = parfait.
Dice2|P∩G| / (|P|+|G|)Similaire à l'IoU, pénalise moins les petits objets.
Pixel Accuracy(VP+VN) / NProportion globale de pixels bien classés.
Boundary F1F1 sur contours (tolérance 2 px)Qualité des bords du masque.

Métriques d'inpainting

MétriqueRôle
PSNR (Peak Signal-to-Noise Ratio)Fidélité pixel-par-pixel sur la zone inpaintée. ↑ mieux.
SSIM (Structural Similarity Index)Préservation des structures visuelles. ↑ mieux (max 1).
LPIPSPerceptual similarity via features profondes. ↓ mieux.

Protocole de benchmark

Un jeu de données synthétique est généré par generate_synthetic_eval_dataset.py : pour chaque image, un objet connu est collé sur un fond enregistré, ce qui fournit la vérité terrain exacte du masque et du fond original. Les scripts evaluate_segmentation.py et evaluate_inpainting.py calculent les métriques sur l'ensemble du jeu et produisent des tableaux CSV + graphiques comparatifs.

8. Conclusion & Perspectives

Bilan

Le projet implémente avec succès un pipeline de segmentation interactive robuste basé sur GrabCut, avec un raffinement morphologique automatique, et un système d'inpainting hybride (LaMa + fallback TELEA) produisant des reconstructions de fond visuellement convaincantes. L'interface PySide6 rend l'outil accessible et ergonomique.

✅ Points forts

  • Architecture modulaire et testée (pytest)
  • Support de 3 modes d'annotation
  • Inpainting neuronal de haute qualité (LaMa)
  • Feathering automatique des bords
  • Pipeline d'évaluation quantitatif complet

🔮 Perspectives

  • Intégration de SAM (Segment Anything Model)
  • Inpainting par diffusion (Stable Diffusion)
  • Exportation en PNG avec canal alpha
  • Mode vidéo (segmentation par propagation)
  • Interface web (Flask / FastAPI + React)
